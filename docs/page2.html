Ámbito de las variables

El ámbito de una variable (llamado "scope" en inglés) es la zona del programa en la que se define la variable.
JavaScript define dos ámbitos para las variables: global y local.

Veamos un ejemplo:

      function unMensaje() {
        var mensaje = “Mostrar mensaje”;
      }
      unMensaje();
      alert(mensaje);

En primer lugar tenemos una función llamada unMensaje que crea una variable llamada mensaje.
A continuación, se ejecuta la función mediante unMensaje(); y luego, se muestra mediante alert() el valor de mensaje.

Sin embargo, al ejecutar el código anterior no se muestra ningún mensaje por pantalla. La razón es que la variable mensaje se ha definido dentro de la función unMensaje() y por tanto, es una variable local que solamente está definida dentro de la función.


Cualquier instrucción que se encuentre dentro de la función puede hacer uso de esa variable, pero todas las instrucciones que se encuentren en otras funciones o fuera de cualquier función no tendrán definida la variable mensaje.

De esta forma, para mostrar el mensaje en el código en cuestión, la función alert() debe llamarse desde dentro de la función unMensaje():

      function unMensaje() {
        var mensaje = “Mostrar mensaje”;
        alert(mensaje);
      }
      unMensaje();

Es importante saber que...Además de variables locales, también existe las variables globales, que pueden ser usadas en cualquier punto del programa (incluso dentro de cualquier función).

      var elMensaje = “Mensaje de prueba”;
       
      function muestraMensaje() {
        alert(elMensaje);
      }

Este código es inverso al mostrado primero, ya que dentro de la función muestraMensaje() se hace uso de una variable llamada elMensaje que no ha sido definida dentro de la propia función. Sin embargo, si se ejecuta el código, sí que se muestra el el contenido definido por la variable elMensaje.

Esto es por que, la variable elMensaje se ha definido fuera de cualquier función. Este tipo de variables automáticamente se transforman en variables globales y están disponibles en cualquier punto del programa (incluso dentro de cualquier función).

De esta forma, aunque en el interior de la función no se ha definido ninguna variable llamada elMensaje, la variable global creada anteriormente permite que la instrucción alert() dentro de la función se muestre correctamente.

Importante: Si una variable se declara fuera de cualquier función, automáticamente se transforma en variable global independientemente de si se define utilizando la palabra reservada var o no. Sin embargo, las variables definidas dentro de una función pueden ser globales o locales.

Si en el interior de una función, las variables se declaran mediante var se consideran locales y las variables que no se han declarado mediante var, se transforman automáticamente en variables globales.

Por lo tanto, se puede rehacer el código del primer ejemplo para que muestre el mensaje correctamente. Para ello, simplemente se debe definir la variable dentro de la función sin la palabra reservada var, para que se transforme en una variable global:

      function unMensaje() {
        mensaje = "Mostrar mensaje";
      }
       
      unMensaje();
      alert(mensaje);

¿Qué sucede si una función define una variable local con el mismo nombre que una variable global que ya existe? En este caso, las variables locales prevalecen sobre las globales, pero sólo dentro de la función.

La recomendación general es definir como variables locales todas las variables que sean de uso exclusivo para realizar las tareas encargadas a cada función. Las variables globales se utilizan para compartir variables entre funciones de forma sencilla.

Lo básico

Un array es una colección de variables, que pueden ser todas del mismo tipo o cada una de un tipo diferente. Su utilidad se comprende mejor con un ejemplo sencillo... Si una aplicación necesita manejar los países de Suramérica, se podrían crear doce variables de tipo texto:

      var pais1 = "Argentina";
      var pais2 = "Bolivia";
      ...
      var pais12 = "Venezuela";

Este código no es incorrecto, pero si poco eficiente y complica en exceso la programación. Si en vez de los países se tuviera que guardar el nombre de de todos los países del mundo o las mediciones diarias de marea de los últimos 100 años, se tendrían que crear decenas o cientos de variables.

En este tipo de casos, se pueden agrupar todas las variables relacionadas en una colección de variables o array. El ejemplo anterior se puede rehacer de la siguiente forma:


      var paises = ["Argentina" , "Bolivia" , "Brasil" ,
      "Chile" , "Colombia" , "Ecuador" , "Guyana" , "Paraguay" ,
      "Perú" , "Surinam" , "Uruguay" , "Venezuela"];

Ahora, una única variable llamada paises almacena todos los valores relacionados entre sí, en este caso los países de Suramérica.

Para definir un array, se utilizan los caracteres [ y ] para delimitar su comienzo y su final y se utiliza el carácter , (coma) para separar sus elementos:


      var nombre_array = [valor1, valor2, ..., valorN];

Acceder elementos de un array

Una vez definido un array, es muy fácil acceder a cada uno de sus elementos.

A cada uno se accede indicando su posición en el array. La único que debemos tener en consideración, que es responsable de muchos errores cuando se empieza a programar, es que las posiciones de los elementos empiezan a contarse en el 0 y no en el 1:


      var paises = ["Argentina" , "Bolivia" , "Brasil" ,
      "Chile" , "Colombia" , "Ecuador" , "Guyana" , "Paraguay" ,
      "Perú" , "Surinam" , "Uruguay" , "Venezuela"];

      var primerPais = paises[0];         // primerPais = "Argentina"
      var paisSeleccionado = paises[10];  // paisSeleccionado = "Uruguay"
      var paisCafetero = paises[4];       // paisCafetero = "Colombia"

En el ejemplo, la primera instrucción quiere obtener el primer elemento del array. Para eso, se indica el nombre del array y entre corchetes la posición del elemento dentro del array. Como se ha comentado, las posiciones se empiezan a contar en el 0, por lo que el primer elemento ocupa la posición 0 y se accede a el mediante paises[0].

El valor paises[10] hace referencia al elemento que ocupa la onceava posición dentro del array paises el valor Uruguay. Como las posiciones empiezan a contarse en 0, la posición 4 hace referencia al quinto elemento, en este caso, el valor Colombia.

Si deseamos podemos cambiar en valor de un elemento de un array, accediendo a él con el mismo procedimiento ya descrito.


      var numeros = [ 1 , 2 , 3 ];
      console.log(numeros);    // Muestra esto [1, 2, 3]
      console.log(numeros[1]); // Muestra 2
      numeros[0] = "primero";  // Cambia el valor
      console.log(numeros[0]); // Muestra esto "primero"

Un array puede contener variables de varios tipos (string, números, booleanos, etc.) sin nigún problema.


Métodos de arrays o matrices

A continuación se muestran algunas de las funciones más útiles para el manejo de arrays:


shift()

Elimina el primer elemento del array y lo devuelve. El array original se modifica y su longitud disminuye una unidad.

      var array = [1, 2, 3];
      var primero = array.shift();
      // ahora array = [2, 3]
      console.log(primero); // Muestra 1

pop()

Elimina el último elemento del array y lo devuelve. El array original se modifica y su longitud disminuye en 1 elemento.

      var array = [1, 2, 3];
      var ultimo = array.pop();
      // ahora array = [1, 2], ultimo = 3

unshift()

añade un elemento al principio del array. El array original se modifica y aumenta su longitud en 1 elemento. (También es posible añadir más de un elemento a la vez)

      var array = [1, 2, 3];
      array.unshift(0);
      // ahora array = [0, 1, 2, 3]

push()

Añade un elemento al final del array. El array original se modifica y aumenta su longitud en 1 elemento. (También es posible añadir más de un elemento a la vez)

      var array = [1, 2, 3];
      array.push(4);
      // ahora array = [1, 2, 3, 4]

reverse()

Modifica un array colocando sus elementos en el orden inverso a su posición original:

      var array = [1, 2, 3];
      array.reverse();
      // ahora array = [3, 2, 1]

splice()

Quita elementos de una matriz, inserta nuevos elementos en su lugar si procede y devuelve los elementos eliminados.

Sintaxis:

array.splice(start, deleteCount, [item1[, item2[, . . . [,itemN]]]])

array Array sobre el que se actua, es obligatorio.

start Inicio (basado en cero de la matriz) desde donde va a comenzar a quitar/agregar elementos, es obligatorio.

deleteCount Número de elementos que se va a quitar, si se deja en 0 no elimina nada, por lo que este ,étodo funciona en este caso para añadir elementos. Es obligatorio que figure una cifra, así sea 0.

item1, item2,. . ., itemN Elementos que se van a insertar en la matriz en lugar de los elementos eliminados. Es opcional.

Ejemplo eliminando elementos:

      var array = [1, 2, 3, 4, 5];
      array.splice( 1, 2 );
      // ahora array = [1, 4, 5]

Ejemplo añadiendo elementos:

      var array = [1, 4, 5];
      array.splice( 1, 0, 2, 3 );
      // ahora array = [1, 2, 3, 4, 5]

indexOf()

Calcula la primera posición en la que se encuentra el elemento indicado dentro del array. Si no contiene el elemento, la función devuelve el valor -1:

      var array = [1, 2, 3, 4, 5];
      var posicion = array.indexOf(3); // posicion = 2
      posicion = array.indexOf(7); // posicion = -1

Arrays Multidimencionales

Veamos qué son los arrays multidimensionales (arrays de más de una dimensión) y cómo utilizarlos. Además de cómo inicializar arrays en su declaración.

Los arrays son bastante importantes en Javascript y también en la mayoría de los lenguajes de programación. En concreto ya hemos aprendido a crear arrays y utilizarlos. Pero aun nos quedan algunas cosas importantes, como son los arrays de varias dimensiones.

Los arrays multidimensionales son un estructuras de datos que almacenan los valores en más de una dimensión. Los arrays que hemos visto hasta ahora almacenan valores en una dimensión, por eso para acceder a las posiciones utilizamos tan solo un índice. Los arrays de 2 dimensiones guardan sus valores, por decirlo de alguna manera, en filas y columnas y por ello necesitaremos por lo menos dos índices para acceder a cada una de sus posiciones.

Dicho de otro modo, un array multidimensional es como un contenedor que guardara más valores para cada posición, es decir, los elementos del array son a su vez otros arrays.

Javascript tiene una forma resumida de construir un Array Multidimencional, para ello simplemente escribimos entre corchetes los datos del array que estamos creando.

A continuación un ejemplo sobre cómo utilizar esta sintaxis para declarar arrays de más de una dimensión.


      var arrayMuchasDimensiones = [
        1, 
        [
          "hola", 
          "que", 
          "tal", 
          [
            "estas", 
            "estamos", 
            "estoy"
          ], 
          [
            "bien", 
            "mal"
          ], 
          "acabo"
        ], 
        2, 
        5
      ];

Así hemos creado un array que tiene casillas con contenido de simples enteros y otras con contenido de cadena y otras que son otros arrays. Podríamos acceder a algunas de sus casillas y mostrar sus valores de esta manera:


      alert (arrayMuchasDimensiones[0])        // Esto es igual a 1
      alert (arrayMuchasDimensiones[1][2])     // Esto es igual a "tal"
      alert (arrayMuchasDimensiones[1][3][1])  // Esto es igual a "estamos"

      Expresiones Funcionales

      Vamos a ver cómo recorrer un array en Javascript, a través de tres métodos, muy usados.

      Una de las partes más potentes, y a menudo confusas, de JavaScript es el manejo de las funciones.

      Existen dos formas de definir una función:

      Mediante declaración:

      
      function nombreFuncion(arg0,arg1,arg2) {
        // cuerpo de la función
      }

      Las funciones creadas mediante declaración o Funciones Declaradas pueden aparecer en el código luego de una llamada a dicha función, es decir pueden ser llamadas antes de ser declaradas.

      Mediante expresión:

      
      var nombreFuncion = function(arg0,arg1,arg2) {
        // cuerpo de la función
      }

      Una función creada mediante expresión se considera una Función Anónima (o función lambda), que se asigna a una variable. Este tipo de expresión es igual a cualquier otra, y por tanto, deben asignarse antes del uso. Si la llamada ocurre antes de la asignación, el programa fallará. La función aún no existe, aunque, debido al modo en que el motor de JavaScript parsea el código, que “eleva” a la parte de arriba la declaración de variables –hoisting– la variable sí existirá, pero su valor será todavía undefined.

      Este método de definición permite hacer cosas así:

      
      var saludar;
       
      if(temprano) {
            saludar = function() { alert("Buenos días"); };
      } else {
            saludar = function() { alert("Buenas tardes"); };
      }

      También es posible que las funciones devuelvan otras funciones. Por ejemplo, se puede crear una función que devuelva funciones comparadoras personalizadas según cierta propiedad:

      
      function crearFuncionComparadora(propiedad) {

          return function(obj1, obj2) {

              var valor1 = obj1[propiedad];
              var valor2 = obj2[propiedad];

              if (valor1 < valor2) {
                  return -1;
              } else if (valor1 > valor2) {
                  return 1;
              } else {
                  return 0;
              }
          };
      }

      Hoisting (Alzado)

      Hoisting es un término que no encontrará utilizado en ninguna especificación previa a ECMAScript® 2015 Language Specification.

      El concepto de Hoisting fue pensado como una manera general de referirse a cómo funcionan los contextos de ejecución en JavaScript (específicamente las fases de creación y ejecución).

      Hoisting puede llevar a malos entendidos, por ejemplo, puede dar a entender que las declaraciones de variables y funciones son físicamente movidas al comienzo del código, pero esto no es lo que ocurre en realidad. Lo que sucede es que las declaraciones de variables y funciones son asignadas en memoria durante la fase de compilación, pero quedan exactamente en dónde la ha escrito en el código.

      Ejemplo

      Una de las ventajas en JavaScript de colocar (ponerlas en memoria) las declaraciones de funciones antes de ejecutar cualquier otro segmento de código es que permite utilizar una función antes de declararla en el código:

      
      function nombreDelGato(nombre) {
        console.log("El nombre de mi gato es " + nombre);
      }

      nombreDelGato("Maurizzio");
      /*
      El resultado del código es: "El nombre de mi gato es Maurizzio"
      */

      El código escrito arriba está escrito de la manera que sería esperada para que funcione.

      Ahora, veamos qué sucede cuando llamamos a la función antes de escribirla:

      
      nombreDelGato("Dumas");

      function nombreDelGato(nombre) {
        console.log("El nombre de mi gato es " + nombre);
      }
      /*
      El resultado del código es: "El nombre de mi gato es Dumas"
      */

      Como se puede observar, aunque primero llamamos a la función en el código, antes de que sea escrita, el código aún funciona. Esto sucede por la manera en la que el contexto de ejecución trabaja en JavaScript.


      Fuente: https://developer.mozilla.org/es/docs/Glossary/Hoisting/

      Closures (Clausuras)

      Es una función que retorna desde otra función, y la clave está en que lleva una copia o una referencia del ámbito de esa función que la retorna, es decir encierra o envuelve todo el ambito con las variables o las funciones que hubiera y se lo lleva con ella.

      Otro concepto válido es: "Un closure o clausura es la combinación de una función y el ámbito léxico en el que se declaró dicha función. Es decir los closures o clausuras son funciones que manejan variables independientes. En otras palabras, la función definida en el closure "recuerda" el ámbito en el que se ha creado".

      Veamos lo siguiente:

      
      function inicia() {
        var nombre = "Mozilla"; // 'nombre' es una variable local creada por la función 'inicia'
        function muestraNombre() { // 'muestraNombre' es una función interna (un closure)
          alert(nombre); // dentro de esta función usamos una variable declarada en la función padre
        }
        muestraNombre();
      }
      inicia();

      La función inicia()  crea una variable local llamada nombre.
      A continuación, define una función denominada muestraNombre().
      muestraNombre() es una función interna (un closure) definida dentro de inicia(), y sólo está disponible en el cuerpo de esa función.
      muestraNombre() no tiene ninguna variable propia, lo que hace es reutilizar la variable nombre declarada en la función externa.

      Esto funciona bien, intenta correr el código para ver qué pasa. Este es un ejemplo de ámbito funcional: en JavaScript, el ámbito de una variable se define por su ubicación dentro del código fuente y las funciones anidadas tienen acceso a las variables declaradas en su ámbito externo.

      Objetos

      Un objeto es una colección de propiedades, y una propiedad es una asociación entre un nombre y un valor. Un valor de propiedad puede ser una función, la cual es conocida entonces como un método del objeto.

Comparado con un libro por ejemplo, un libro es un objeto con propiedades, tiene un autor, un título, un género, etc, de la misma manera, los objetos de JavaScript pueden tener propiedades, que definen sus características.

          Métodos

          Las cosas que pueden hacer los objetos se llaman métodos. Las personas hablan, las computadoras se rompen y las bicicletas ruedan.

          Los objetos de JavaScript también tienen métodos: click() para los botones, open() para las ventanas y el texto puede seleccionarse gracias al método selected(). Los paréntesis indican que estamos refiriéndonos a un método, y no a una propiedad.

          Consejo, puede ser útil pensar en los objetos y en las propiedades como nombres, y en los métodos como verbos. Los primeros son cosas, y los segundos, acciones que esas cosas pueden hacer, o que se les pueden hacer a ellas.

          Puede poner juntos los objetos, las propiedades y los métodos para obtener una descripción mejor de un objeto, o para describir un proceso.

          Prototipado y herencia en JavaScript

          JavaScript puede resultar un poco extraño a desarrolladores con experiencia en lenguajes basados en clases (como Java o C++), ya que es más dinámico y no proporcionar una implementación de clases en sí mismo (la palabra clave class se introdujo en ES2015, pero sólo para endulzar la sintaxis, ya que JavaScript sigue estando basado en prototipos).

          En lo que a herencia se refiere, JavaScript sólo tiene una estructura: objetos ... Y ya lo hemos visto antes, en este lenguaje todos son objetos, y todo quiere decir que los números son objetos, las funciones también son objetos y las "clases" que crearán objetos también son objetos.

          Una vez que se ha entendido esta idea es fácil entender cosas como las funciones anónimas o el prototipado, manos a la obra.

          Creación de clases

          Primero que nada creamos nuetra Clase.

          
      function Persona (edad) {
        this.edad = edad || 0;
      }

      var veronica = new Persona(6),
          carlos = new Persona(12);

      console.log(veronica instanceof Persona); 
      // Muestra: true
      console.log(carlos.edad); 
      // Muestra: 12

          En vista de esto, definimos nuestro constructor, tarea que desempeñará una función que hará uso de la variable this como el objeto creado.

          Desde el momento en el que se defina la función podemos crear instancias de la clase para obtener nuevos objetos.

          El paso siguiente es añadir métodos que puedan ser utilizados por los objetos creados, y dado que en JavaScript todo es un objeto podemos hacer algo así:

          
      function Persona (edad) {
        this.edad = edad;
        this.crecer = function () {
          this.edad = this.edad + 1;
          return this.edad;
        }
      }

      var veronica = new Persona(6);
      console.log(veronica.crecer()); 
      // Muestra: 7

          En efecto el código anterior funciona y da una forma de implementar métodos de instancia, pero hay un detalle importante, nuestro constructor está creando una función crecer para cada una de las instancias que creamos de la clase, por lo tanto si tenemos un array con 10.000 instancias de Persona se crearán 10.000 funciones distintas (e independientes), algo que en términos de eficiencia es patético.

          Afortunadamente, JavaScript tiene un atributo muy últil para todos los objetos: prototype. Este atributo es en si es un objeto vacío; pero que posee unas características muy interesantes. Cuando una función se utiliza como constructor (que es lo que estábamos haciendo en los ejemplos anteriores) JavaScript establece como atributo prototype del objeto al mismo prototipo del constructor, de forma que los objetos creados pueden acceder a todos los atributos que se encuentren dentro de este objeto sin necesidad de duplicar información.

          Veamos como queda el código con este concepto:

          
      function  Persona (edad) {
        this.edad = edad;
      }

      Persona.prototype.crecer = function () {
        this.edad = this.edad + 1;
        return this.edad;
      };

          Construir objetos en JavaScript

          Para JavaScript, como lenguaje de programación, casi TODO es un objeto. Números, cadenas, arrays, funciones, datos booleanos, fechas y expresiones regulares son percibidas como objetos o pueden ser tratados como tal.

          Para entender mejor el concepto de objeto imaginemos que acabamos de comprar un móvil nuevo (el móvil es el objeto). Tiene propiedades como color, peso, tamaño, tamaño de pantalla y marca; también tiene métodos como llamar, tomar una foto, encendido y apagado. Ahora, expliquémoslo a nivel de programación:

          • Los objetos son una instancia de una clase
          • Las propiedades son una característica, dada en un valor; ya sea tipo String (cadena), matemática, etc
          • Los métodos son una capacidad; y para realizarlos utilizamos una función (function)

          En JavaScript encontramos 3 maneras de crear objetos:

          Crear un objeto literal Crear un objeto usando la palabra new Crear un objeto haciendo una función constructora

          Crear un objeto literal

          Un objeto literal es la manera más simple de crear un objeto. Se enmarca dentro de corchetes y se describe en pares de (clave: valor).

        Marca : "Alcatel", 
        Peso : "5 onzas", 
        pantalla : "5 pulgadas", 
        Color : "Negro"
      };

          Crear un objeto usando la palabra new

          El operador new crea una instancia de un tipo de objeto a partir de una función constructora. Recordemos que en JavaScript un constructor es una función.

          
      var movil = newObject();
      movil.Marca = "Alcatel";
      movil.Peso = "5 onzas";
      movil.Pantalla = "5 pulgadas";
      movil.Color = "Negro";

          ¿Qué es el DOM?

          El modelo de objeto de documento (DOM) es una interfaz de programación para los documentos HTML y XML, que facilita una representación estructurada del documento y define de qué manera los programas pueden acceder, al fin de modificar, tanto su estructura, estilo y contenido.

          El DOM da una representación del documento como un grupo de nodos y objetos estructurados que tienen propiedades y métodos.

          Esencialmente, conecta las páginas web a scripts o lenguajes de programación.

          La especificación completa de DOM define 12 tipos de nodos, aunque las páginas XHTML habituales se pueden manipular manejando solamente cuatro o cinco tipos de nodos:

          • Document, nodo raíz del que derivan todos los demás nodos del árbol.
          • Element, representa cada una de las etiquetas XHTML. Se trata del único nodo que puede contener atributos y el único del que pueden derivar otros nodos.
          • Attr, se define un nodo de este tipo para representar cada uno de los atributos de las etiquetas XHTML, es decir, uno por cada par atributo=valor.
          • Text, nodo que contiene el texto encerrado por una etiqueta XHTML.
          • Comment, representa los comentarios incluidos en la página XHTML.

          Los otros tipos de nodos existentes que no se van a considerar son DocumentType, CDataSection, DocumentFragment, Entity, EntityReference, ProcessingInstruction y Notation.

          Como sucede con todas las funciones del DOM, getElementsByTagName() tiene un nombre muy largo, pero que lo hace explicarse por si mismo.

          Añadir Nodos

          Un elemento sencillo, como por ejemplo un párrafo, genera dos nodos: el primer nodo es de tipo Element y representa la etiqueta <p> y el segundo nodo es de tipo Text y representa el contenido textual de la etiqueta <p>.

          Por eso, crear y añadir a la página un nuevo elemento consta de cuatro pasos diferentes:

          1. Creación de un nodo de tipo Element que represente al elemento.
          2. Creación de un nodo de tipo Text que represente el contenido del elemento.
          3. Añadir el nodo Text como nodo hijo del nodo Element.
          4. Añadir el nodo Element a la página, en forma de nodo hijo del nodo correspondiente al lugar en el que se quiere insertar el elemento.

          De este modo, si se quiere añadir un párrafo al final de una sección, es necesario incluir el siguiente código:

          
      // Crear variable para seleccionar sección
      var seccion = document.getElementById("contParrafos");

      // Crear nodo de tipo Element
      var parrafo = document.createElement("p");
       
      // Crear nodo de tipo Text
      var contenido = document.createTextNode("Yo soy un 
      párrafo creado con JavaScrip");
       
      // Añadir el nodo Text como hijo del nodo Element
      parrafo.appendChild(contenido);
       
      // Añadir el nodo Element como hijo de la sección
      seccion.appendChild(parrafo);
